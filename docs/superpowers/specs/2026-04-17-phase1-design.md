# 网盘项目第一期设计文档

## 1. 目标

第一期实现一个可运行的 C 语言网盘基础版本，同时开发服务端和客户端，完成以下核心流程：

- 客户端连接服务端
- 客户端使用用户名进行伪登录
- 服务端为不同用户隔离目录
- 客户端执行 `pwd`、`ls`、`cd`、`mkdir`、`rm`、`puts`、`gets`、`quit`
- 服务端记录连接、登录、命令、文件传输等日志
- 项目支持通过配置文件加载 IP、端口、线程数、日志级别、数据目录等信息

第一期只实现基础命令交互和普通文件传输，不实现第二期及之后的断点续传、`mmap`、数据库、真实用户系统等功能。

## 2. 总体架构

项目根目录直接使用当前工作目录，不再额外套一层 `mycloud/`。正式代码与 `reference_code/` 分离。

服务端复用参考代码的总体思路：

- 主线程负责监听和接收新连接
- 主线程将新连接放入任务队列
- 工作线程从任务队列中取出 `client_fd`
- 每个工作线程负责一个客户端连接的完整会话，直到客户端断开

客户端与服务端之间使用简单文本协议：

- 文本命令：先发送字符串长度，再发送字符串内容
- 文本响应：先发送字符串长度，再发送字符串内容
- 文件上传下载：在文本命令确认后，再发送或接收文件大小与文件内容

## 3. 多用户与虚拟路径

第一期需要支持多用户伪登录，但不做真实认证。用户信息放在 `config/users.conf` 中，每行一个用户名。

服务端启动后读取用户列表。客户端连接成功后，必须先发送 `login 用户名`。服务端检查用户名是否存在：

- 存在：登录成功，为该连接初始化会话
- 不存在：登录失败，不允许执行普通命令

每个用户都映射到独立的真实目录，例如：

- `alice` -> `./data/users/alice`
- `bob` -> `./data/users/bob`

客户端看到的始终是虚拟路径，例如 `/`、`/dir1`、`/dir1/dir2`，不会看到真实物理路径。

## 5. 协议设计

### 5.1 文本命令

客户端发送一条命令时，格式如下：

1. 发送 `int` 类型长度
2. 发送命令字符串

例如：

- `login alice`
- `pwd`
- `ls`
- `cd dir1`

### 5.2 文本响应

服务端响应文本时，格式如下：

1. 发送 `int` 类型长度
2. 发送响应字符串

响应建议使用简单的文本状态：

- `OK ...`
- `ERROR ...`

### 5.3 文件上传下载

上传流程：

1. 客户端发送 `puts 文件名`
2. 服务端检查当前状态与目标路径，返回是否允许上传
3. 客户端发送文件大小
4. 客户端发送文件内容

下载流程：

1. 客户端发送 `gets 文件名`
2. 服务端检查文件是否存在
3. 服务端发送文本状态
4. 服务端发送文件大小
5. 服务端发送文件内容

## 6. 会话结构

每个客户端连接在服务端都有一份会话信息：

```c
typedef struct Session {
    int client_fd;
    int is_login;
    char user_name[64];
    char current_path[512];
    char user_root[512];
} Session;
```

作用如下：

- `client_fd`：当前客户端套接字
- `is_login`：是否已经登录
- `user_name`：当前用户名
- `current_path`：当前虚拟路径
- `user_root`：当前用户真实根目录

## 7. 第一期开启的命令

- `login 用户名`
- `pwd`
- `ls`
- `cd 目录名`
- `cd ..`
- `mkdir 目录名`
- `rm 文件名`
- `puts 文件名`
- `gets 文件名`
- `quit`

范围约束：

- 登录前只允许 `login` 和 `quit`
- `cd` 只支持当前目录下一级目录，以及 `cd ..`
- 不支持一次输入复杂路径，如 `cd a/b/c`
- `rm` 第一阶段只删除普通文件，不删除目录
- `puts` 与 `gets` 只处理普通文件

## 8. 模块划分

### 8.1 配置模块

负责读取 `config/mycloud.conf` 和 `config/users.conf`，并提供获取配置项与检查用户名的接口。

### 8.2 日志模块

负责初始化日志文件，按照日志级别输出日志到终端和日志文件。

### 8.3 协议模块

负责封装长度加内容的文本收发，以及文件大小等协议辅助函数。

### 8.4 会话模块

负责初始化会话、拼接真实路径、处理虚拟路径变化，保证用户不能越界访问其他用户目录。

### 8.5 命令处理模块

负责解析命令、校验参数、分发到具体命令处理函数。

### 8.6 文件传输模块

负责处理第一期的普通上传与下载逻辑。

### 8.7 线程池与队列模块

负责复用参考代码中的生产者消费者模型，完成新连接的调度。

### 8.8 客户端模块

负责建立连接、登录、读取命令、发送请求、接收响应、执行文件上传下载。

## 9. 错误处理

第一期采用简单直接的错误处理方式：

- 非法命令：返回 `ERROR invalid command`
- 未登录执行普通命令：返回 `ERROR please login first`
- 用户不存在：返回 `ERROR user not found`
- 目录不存在：返回 `ERROR directory not exist`
- 文件不存在：返回 `ERROR file not exist`
- 删除目录：返回 `ERROR cannot remove directory`
- 网络读写失败：关闭当前连接

## 10. 日志策略

第一期日志只在服务端实现，记录：

- 服务端启动
- 客户端连接与断开
- 登录成功与失败
- 命令执行
- 上传下载结果
- 关键错误

日志级别分为：

- `ERROR`
- `INFO`
- `DEBUG`

## 11. 验证方式

第一期主要采用人工联调验证：

1. 启动服务端
2. 启动客户端
3. 使用已存在用户名登录
4. 依次验证 `pwd`、`ls`、`mkdir`、`cd`、`puts`、`gets`、`rm`、`quit`
5. 验证非法命令与异常路径

该验证流程会同步写入项目文档，作为第一期交付说明的一部分。
