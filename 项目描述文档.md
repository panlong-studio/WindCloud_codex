# 开发规范

*   **函数名**：模块名作为前缀 + 动作。
    *   `config_load()`、`log_write()`、`protocol_send_string()`
*   **变量名**：简洁且有意义。
    *   `int listen_fd;`、`int client_fd;`
    *   `char user_name[64];`
*   **宏与常量**：全大写 + 下划线。
    *   `#define MAX_BUFFER_SIZE 1024`
    *   `#define LOG_LEVEL_INFO 1`
*   **结构体**：统一使用 `_t` 后缀。
    *   `typedef struct session_s session_t;`
    *   `typedef struct thread_pool_s thread_pool_t;`
*   **代码风格**：
    *   第一期开发尽量使用基础语法，不使用过于复杂的高级写法。
    *   每个函数都要写清楚功能、入参、返回值。
    *   逻辑复杂的位置要加必要注释，便于初学者阅读。

# 详细功能需求

## 第一期：基础框架与核心命令交互

### 1、客户端需求 (Client)

- **长连接交互**：
  - 客户端启动后与服务端建立长连接。
  - 登录成功后，在终端提供简单命令提示符。
- **伪登录功能**：
  - 客户端启动后，先输入用户名。
  - 客户端向服务端发送 `login 用户名`。
  - 如果用户名存在，则进入命令循环；否则提示登录失败。
- **命令输入与解析**：
  - 循环读取标准输入。
  - 使用 `fgets` + `sscanf` 或 `strtok` 解析命令和参数。
  - 过滤空命令和无效命令。
- **支持的基础命令**：
  - `pwd`：显示当前所在虚拟路径。
  - `ls`：列出当前目录下的文件和文件夹。
  - `cd <dir>`：进入当前目录下的一级子目录。
  - `cd ..`：返回上一级目录。
  - `mkdir <dir>`：在当前目录下创建新文件夹。
  - `rm <file>`：删除服务器上的普通文件。
  - `puts <file>`：将本地文件上传到服务器当前目录。
  - `gets <file>`：将服务器当前目录中的文件下载到本地 `download/` 目录。
  - `quit`：主动断开连接。

### 2、服务端需求 (Server)

- **配置文件动态加载**：
  - 启动时读取 `config/mycloud.conf`。
  - 第一期开启的配置项包括：IP、端口、线程数、日志级别、日志文件路径、数据根目录。
- **伪登录与多用户隔离**：
  - 启动时读取 `config/users.conf`。
  - 每行一个用户名。
  - 服务端根据用户名为不同用户分配独立目录，例如 `./data/users/alice`。
- **会话状态管理**：
  - 服务端需要为每个连接维护一份会话信息。
  - 会话中至少包含：客户端 fd、是否登录、用户名、当前虚拟路径、用户真实根目录。
- **虚拟路径隔离**：
  - 客户端看到的始终是虚拟路径，例如 `/`、`/dir1`。
  - 服务端内部把虚拟路径映射到真实目录，例如 `./data/users/alice/dir1`。
  - 严禁客户端越界访问其他用户目录。
- **命令处理与文件操作**：
  - 服务端接收客户端发送的命令字符串。
  - 解析命令后，执行对应文件或目录操作。
  - 第一阶段只处理普通文件与基础目录切换，不处理断点续传、数据库、用户认证等后续功能。
- **自定义日志系统**：
  - 记录服务端启动信息。
  - 记录客户端连接和断开。
  - 记录登录成功与失败。
  - 记录命令执行结果。
  - 记录上传下载和错误信息。

### 3、第一期明确不实现的内容

- 第二期的大文件断点续传
- 第二期的 `mmap` 大文件优化
- 第三期数据库
- 真实密码认证
- 跨多级复杂路径解析
- 递归删除目录

## 第二期：大文件传输优化与断点续传

以下内容属于后续阶段，本期不实现，只保留说明：

### 1、断点续传 (Resumable Transfer)

- **上传 (puts)**：
  - 网络中断后再次上传同一文件时，客户端先向服务器发送文件名。
  - 服务器检测该文件是否已存在及当前大小。
  - 客户端跳过已上传的部分，从断点处继续发送；服务端从断点处追加写入。
- **下载 (gets)**：
  - 客户端本地检测是否存在未下载完的文件，获取本地文件大小。
  - 客户端将大小发送给服务器，服务器从指定偏移量处继续传输剩余数据。

### 2、大文件内存映射 (mmap)

- 当上传或下载的文件大小超过预设阈值时：
  - 接收端可放弃传统 `read/write`。
  - 使用 `ftruncate` + `mmap` 提高大文件处理效率。
